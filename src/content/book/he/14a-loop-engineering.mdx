הנדסת הקשר מחליטה **מה הדגם יכול לראות**. הנדסת לולאות מחליטה **מה המערכת עושה הלאה**.

במקום שאדם יקרא שוב ושוב תשובה ויכתוב את ההנחיה הבאה, לולאה הופכת את עבודת ההמשך למערכת מעוצבת. זה נותן ל-AI מטרה, נותן לו לפעול, צופה במשוב אמיתי, מעריך את התוצאה, או מסתגל או עוצר.

<Callout type="info" title="ההגדרה הקצרה">
הנדסת לולאות היא הפרקטיקה של תכנון מחזור המשוב סביב מערכת בינה מלאכותית: המטרה, הפעולות, התצפיות, ההערכה, המצב, מעקות הבטיחות וחוקי עצירה שמניעים את העבודה לעבר תוצאה הניתנת לאימות.
</Callout>

failure: "אילו מצבים דורשים החלמה או עזרה אנושית?"

<Callout type="warning" title="לולאה ללא יציאה היא מצב כשל">
הגדר תמיד שלוש יציאות: **הצלחה** כאשר הוכחות עומדות ביעד, **כישלון** כאשר השחזור אינו שימושי יותר, ו**התקציב מוצה** כאשר הלולאה מגיעה לגבול העלות או הסיכון המותרים שלה.
</Callout>

1. **בדיקות מדויקות** - סוגים, סכימות, אילוצים, הרשאות, גיבוב
2. **בדיקות ניתנות לביצוע** - בדיקות, ליטרים, סימולציות, אימות קישור
3. **בדיקת חומרה** - מודל עצמאי או אדם המשתמש בקריטריונים בעלי שם
4. **בדיקות תוצאות** - התנהגות משתמש אמיתית או מדדי ייצור לאורך זמן

## צורות לולאה נפוצות

### לולאת התיקון

`שכפל → שנה דבר אחד → הפעל בדיקות → אבחון → חזור או עצור`

הטוב ביותר עבור קוד, תצורה, ניקוי נתונים וכל משימה עם משוב בר הפעלה.

### לולאת ה-Evaluator-Optimizer

`צור → ציון עם רובריקה → החזר משוב ממוקד → עדכון`

הטוב ביותר לכתיבה, תרגום, ביקורת עיצוב ותפוקות שאיכותן משתפרת באמצעות משוב מפורש.

### לולאת המחקר

`חפש → בדוק מקורות → זיהוי פערים או התנגשויות → חפש שוב → סינתזה`

הכי טוב כאשר השלמות אינה ידועה מראש. כלל העצירה צריך למדוד את כיסוי הראיות, לא את מספר תוצאות החיפוש.

### הלולאה המגודרת על ידי האדם

`הכן → אמת אוטומטית → השהה לפני פעולה בסיכון גבוה → אנושי מאשר או הפניה מחדש`

הטוב ביותר עבור תשלומים, פרסום, מחיקה, שינויי גישה, החלטות רפואיות או משפטיות ופעולות תוצאתיות אחרות.

## יישום מינימלי

```typescript
let state = initialize(goal, acceptanceCriteria, budget);

while (state.budget.remaining > 0) {
  const context = assembleRelevantContext(state);
  const proposedAction = await maker.chooseAction(context);
  const observation = await tools.executeWithinPolicy(proposedAction);
  const verdict = await evaluator.check({ goal, observation, state });

  state = recordIteration(state, { proposedAction, observation, verdict });

  if (verdict.status === "passed") return complete(state);
  if (verdict.status === "unsafe" || verdict.status === "blocked") {
    return escalateToHuman(state);
  }

  state = adaptPlan(state, verdict.feedback);
}

return stopWithBudgetReport(state);
```

הקוד הוא החלק הקל. השאלות הקשות הן שאלות תחום: איזו בדיקה מוכיחה שהבאג נעלם? איזה מקור הוא סמכותי? איזו פעולה הפיכה? מי יכול לאשר את השלב האחרון? החלטות אלו הן העבודה האמיתית של הנדסת לולאות.
