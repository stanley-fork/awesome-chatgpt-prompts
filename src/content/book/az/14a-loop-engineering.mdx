Kontekst mühəndisliyi **modelin nəyi görə biləcəyinə** qərar verir. Döngü mühəndisliyi **sistemin bundan sonra nə edəcəyinə** qərar verir.

Bir adamın dəfələrlə cavabı oxuması və növbəti əmri yazması əvəzinə, bir döngə bu təqib işi hazırlanmış bir sistemə çevirir. O, süni intellektə məqsəd qoyur, ona hərəkət etməyə imkan verir, real rəyləri müşahidə edir, nəticəni qiymətləndirir və ya uyğunlaşır, ya da dayandırır.

<Callout type="info" title="Qısa Tərif">
Döngü mühəndisliyi süni intellekt sistemi ətrafında əks əlaqə dövrünün layihələndirilməsi təcrübəsidir: məqsəd, hərəkətlər, müşahidələr, qiymətləndirmə, vəziyyət, qoruyucu barmaqlıqlar və işi yoxlanıla bilən nəticəyə aparan dayandırma qaydaları.
</Callout>

## Əvvəlcə Döngü Müqaviləsini yazın

Model və ya çərçivə seçməzdən əvvəl kiçik bir müqavilə yazın. Bu sahələr qeyri-müəyyəndirsə, dövrə qeyri-müəyyənliyi təkrar xərcə çevirəcəkdir.

```yaml
goal: "Hansı müşahidə edilə bilən vəziyyət doğru olmalıdır?"
inputs: "Döngə nədən başlayır?"
state: "İterasiyalar arasında hansı faktlar və cəhdlər qalır?"
actions: "Sistem hansı alətlərdən istifadə edə bilər və hansı icazələrlə?"
observations: "Bu hərəkətlərdən hansı xarici siqnallar geri qayıdır?"
evaluator: "Nəticəni hansı rubrik və ya deterministik yoxlamalar qiymətləndirir?"
success: "Məqsədin tamamlandığını hansı dəlillər sübut edir?"
failure: "Hansı şərtlər bərpa və ya insan köməyi tələb edir?"
budget: "Maksimum növbə, vaxt, ayələr, pul və ya yan təsirlər"
```

<Callout type="warning" title="Çıxışsız Döngə Uğursuzluq Rejimidir">
Həmişə üç çıxışı müəyyənləşdirin: sübut məqsədə çatdıqda **uğur**, bərpa artıq faydalı olmadıqda **uğursuzluq** və dövrə icazə verilən xərc və ya risk sərhədinə çatdıqda **büdcə tükəndikdə**.
</Callout>

Yoxlayıcının başqa bir AI olması lazım deyil. Mövcud olan ən deterministik qiymətləndiriciyə üstünlük verin:

1. **Dəqiq yoxlamalar** — növlər, sxemlər, məhdudiyyətlər, icazələr, hashlər
2. **İcra edilə bilən yoxlamalar** — testlər, lintrlər, simulyasiyalar, keçidin yoxlanılması
3. **Rubrik yoxlamaları** — müstəqil model və ya adlandırılmış meyarlardan istifadə edən insan
4. **Nəticə yoxlanışı** — zamanla real istifadəçi davranışı və ya istehsal göstəriciləri

</Callout>

## Ümumi Döngə Formaları

### Təmir Döngəsi

`çoxaltmaq → bir şeyi dəyişdirmək → yoxlamaları yerinə yetirmək → diaqnoz qoymaq → təkrar etmək və ya dayandırmaq`

Kod, konfiqurasiya, məlumatların təmizlənməsi və icra edilə bilən rəyi olan istənilən tapşırıq üçün ən yaxşısıdır.

### Qiymətləndirici-Optimallaşdırıcı Döngə

`yaradın → rubrika ilə xal verin → hədəflənmiş rəyi qaytarın → yenidən nəzərdən keçirin`

Yazı, tərcümə, dizayn tənqidi və ifadəli rəy vasitəsilə keyfiyyəti yaxşılaşan nəticələr üçün ən yaxşısı.

### Tədqiqat Döngəsi

`axtarış → mənbələri yoxlayın → boşluqları və ya ziddiyyətləri müəyyənləşdirin → yenidən axtarın → sintez edin'

Tamlıq əvvəlcədən bilinməyəndə ən yaxşısıdır. Dayanma qaydası axtarış nəticələrinin sayını deyil, sübutların əhatə dairəsini ölçməlidir.

### İnsan Qapılı Döngə

`hazırlayın → avtomatik yoxlayın → yüksək riskli fəaliyyətdən əvvəl fasilə verin → insan təsdiq edir və ya yönləndirir'

Ödənişlər, nəşrlər, silinmələr, giriş dəyişiklikləri, tibbi və ya hüquqi qərarlar və digər ardıcıl hərəkətlər üçün ən yaxşısıdır.

## Minimal Tətbiq

```typescript
let state = initialize(goal, acceptanceCriteria, budget);

while (state.budget.remaining > 0) {
  const context = assembleRelevantContext(state);
  const proposedAction = await maker.chooseAction(context);
  const observation = await tools.executeWithinPolicy(proposedAction);
  const verdict = await evaluator.check({ goal, observation, state });

  state = recordIteration(state, { proposedAction, observation, verdict });

  if (verdict.status === "passed") return complete(state);
  if (verdict.status === "unsafe" || verdict.status === "blocked") {
    return escalateToHuman(state);
  }

  state = adaptPlan(state, verdict.feedback);
}

return stopWithBudgetReport(state);
```

Kod asan hissəsidir. Çətin suallar domen suallarıdır: Hansı test səhvin getdiyini sübut edir? Hansı mənbə etibarlıdır? Hansı hərəkət geri qaytarıla bilər? Son addımı kim təsdiqləyə bilər? Bu qərarlar loop mühəndisliyinin faktiki işidir.
