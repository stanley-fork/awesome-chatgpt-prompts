تحدد هندسة السياق **ما يمكن للنموذج رؤيته**. تقرر هندسة الحلقات **ما يفعله النظام بعد ذلك**.

بدلاً من قيام الشخص بقراءة الإجابة بشكل متكرر وكتابة الموجه التالي، تقوم الحلقة بتحويل عمل المتابعة هذا إلى نظام مصمم. فهو يمنح الذكاء الاصطناعي هدفًا، ويتيح له التصرف، ويلاحظ ردود فعل حقيقية، ويقيم النتيجة، وإما أن يتكيف أو يتوقف.

<Callout type="info" title="التعريف القصير">
هندسة الحلقات هي ممارسة تصميم دورة ردود الفعل حول نظام الذكاء الاصطناعي: الهدف، والإجراءات، والملاحظات، والتقييم، والحالة، وحواجز الحماية، وقواعد الإيقاف التي تدفع العمل نحو نتيجة يمكن التحقق منها.
</Callout>

## اكتب عقد الحلقة أولاً

قبل اختيار النموذج أو الإطار، اكتب عقدًا صغيرًا. إذا كانت هذه الحقول غامضة، فإن الحلقة ستحول الغموض إلى تكلفة متكررة.

```yaml
goal: "ما هي الحالة التي يمكن ملاحظتها والتي يجب أن تصبح حقيقية؟"
inputs: "ما الذي يبدأ الحلقة؟"
state: "ما هي الحقائق والمحاولات التي تستمر بين التكرارات؟"
actions: "ما هي الأدوات التي يمكن للنظام استخدامها، وبأي أذونات؟"
observations: "ما هي الإشارات الخارجية التي تعود من تلك التصرفات؟"
evaluator: "ما هي قواعد التقييم أو الشيكات الحتمية التي تحكم النتيجة؟"
success: "ما الدليل الذي يثبت اكتمال الهدف؟"
failure: "ما هي الحالات التي تتطلب التعافي أو المساعدة الإنسانية؟"
budget: "الحد الأقصى للدورات أو الوقت أو الرموز أو المال أو الآثار الجانبية"
```

<Callout type="warning" title="الحلقة بدون مخرج هي وضع فشل">
حدد دائمًا ثلاثة مخارج: **النجاح** عندما تحقق الأدلة الهدف، و**الفشل** عندما لا يعود التعافي مفيدًا، و**استنفاد الميزانية** عندما تصل الحلقة إلى التكلفة المسموح بها أو حدود المخاطرة.
</Callout>

## افصل الصانع عن المدقق

بالنسبة للعمل منخفض المخاطر، يمكن لنموذج واحد أن يقوم بالصياغة والمراجعة الذاتية. بالنسبة للعمل المهم، استخدم مدققًا بوظيفة مختلفة وسلطة محدودة.

<InfoGrid columns={2} items={[
  { label: "صانع", description: "يقترح التغيير، ويستدعي أدوات العمل، ويشرح الأدلة التي ينبغي جمعها.", color: "amber" },
  { label: "فاحص", description: "يتلقى الهدف، والقطعة الأثرية، والدليل؛ يطبق عنوان التقييم؛ لا يمكنه إعادة كتابة العمل الذي يصنفه بصمت.", color: "green" },

لا يجب أن يكون المدقق ذكاءً اصطناعيًا آخر. تفضل المقيم الأكثر حتمية المتاحة:

1. **فحوصات دقيقة** — الأنواع، والمخططات، والقيود، والأذونات، والتجزئة
2. **فحوصات قابلة للتنفيذ** — اختبارات، واختبارات، وعمليات محاكاة، والتحقق من صحة الارتباط
3. **عمليات التحقق من القواعد** — نموذج مستقل أو إنسان يستخدم معايير محددة
4. **التحقق من النتائج** — سلوك المستخدم الحقيقي أو مقاييس الإنتاج بمرور الوقت

<Callout type="tip" title="الثقة ليست دليلا">
لا تزال درجة الثقة التي تم إنشاؤها بواسطة نفس النموذج الذي صنع القطعة الأثرية هي مخرجات النموذج. تعامل معه كتلميح توجيه، وليس كدليل.
</Callout>

## الدولة هي العمود الفقري للحلقة

السياق هو ما يراه النموذج **هذا المنعطف**. الحالة هي السجل المضغوط الذي يسمح للدورة التالية بالاستمرار دون تكرار العمل أو نسيانه.

<Checklist title="حالة حلقة مفيدة" items={[
  { text: "الهدف الحالي ومعايير القبول" },
  { text: "الإجراءات التي تمت تجربتها بالفعل ونتائجها الملحوظة" },
  { text: "القطع الأثرية المنتجة، مع إصدارات أو مراجع مستقرة" },
  { text: "أحكام المقيم والقضايا التي لم يتم حلها" },
  { text: "استهلكت الميزانية ولا تزال الأذونات متاحة" },

قم بتخزين القرارات والأدلة المختصرة، وليس سلسلة الأفكار الخاصة بالنموذج. الحالة الجيدة صغيرة وقابلة للفحص وآمنة للاستئناف.

## أشكال الحلقة المشتركة

### حلقة الإصلاح

"إعادة إنتاج → تغيير شيء واحد → تشغيل عمليات التحقق → تشخيص → تكرار أو إيقاف".

الأفضل للتعليمات البرمجية والتكوين وتنظيف البيانات وأي مهمة ذات تعليقات قابلة للتنفيذ.

### حلقة المقيم والمحسن

"إنشاء ← تسجيل باستخدام نموذج تقييم ← إرجاع الملاحظات المستهدفة ← مراجعة".

الأفضل للكتابة والترجمة ونقد التصميم والمخرجات التي تتحسن جودتها من خلال التعليقات المفصلة.

### حلقة البحث

`بحث ← فحص المصادر ← تحديد الثغرات أو الصراعات ← البحث مرة أخرى ← التركيب

الأفضل عندما لا يكون الاكتمال معروفًا مسبقًا. يجب أن تقيس قاعدة الإيقاف مدى تغطية الأدلة، وليس عدد نتائج البحث.

### حلقة الإنسان المسورة

`التحضير ← التحقق تلقائيًا ← التوقف مؤقتًا قبل اتخاذ إجراء عالي الخطورة ← موافقة الإنسان أو إعادة التوجيه

الأفضل للمدفوعات والنشر والحذف وتغييرات الوصول والقرارات الطبية أو القانونية والإجراءات التبعية الأخرى.

## الحد الأدنى من التنفيذ

```typescript
let state = initialize(goal, acceptanceCriteria, budget);

while (state.budget.remaining > 0) {
  const context = assembleRelevantContext(state);
  const proposedAction = await maker.chooseAction(context);
  const observation = await tools.executeWithinPolicy(proposedAction);
  const verdict = await evaluator.check({ goal, observation, state });

  state = recordIteration(state, { proposedAction, observation, verdict });

  if (verdict.status === "passed") return complete(state);
  if (verdict.status === "unsafe" || verdict.status === "blocked") {
    return escalateToHuman(state);
  }

  state = adaptPlan(state, verdict.feedback);
}

return stopWithBudgetReport(state);
```

الكود هو الجزء السهل. الأسئلة الصعبة هي أسئلة المجال: ما الاختبار الذي يثبت اختفاء الخطأ؟ أي مصدر هو موثوق؟ ما هو الإجراء الذي يمكن التراجع عنه؟ من يمكنه الموافقة على الخطوة النهائية؟ هذه القرارات هي العمل الفعلي لهندسة الحلقات.
